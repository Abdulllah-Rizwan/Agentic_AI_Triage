# Agentic AI Triage API

FastAPI backend. Receives triage payloads from mobile devices, generates SOAP reports using Google ADK agents, pushes real-time updates to the dashboard.

---

## Prerequisites

- Python 3.11 or higher
- Docker and Docker Compose (for PostgreSQL and Redis)
- A Google AI API key (free tier at https://aistudio.google.com)

---

## First-Time Setup

### 1. Start the database and Redis
```bash
# From the project root
docker-compose up -d postgres redis
```

### 2. Create a virtual environment
```bash
cd apps/api
python -m venv .venv

# Activate it — you must do this every time you open a new terminal
source .venv/bin/activate          # Mac / Linux
.venv\Scripts\activate             # Windows
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Set up environment variables
```bash
cp .env.example .env
# Open .env and fill in your GOOGLE_API_KEY
```

### 5. Generate protobuf bindings
```bash
# From the project root
protoc --python_out=apps/api/app/proto/ proto/triage.proto
```

### 6. Run database migrations
```bash
alembic upgrade head
```

---

## Running the Server

```bash
# Make sure your virtual environment is active
source .venv/bin/activate

# Start the API server (auto-reloads on file changes)
uvicorn app.main:socket_app --reload --port 3001
```

The API is now available at: http://localhost:3001

Interactive API docs (auto-generated by FastAPI): http://localhost:3001/docs

---

## Running the Background Worker

The SOAP generation happens in a background worker (Celery). You need this running alongside the API server. Open a second terminal:

```bash
cd apps/api
source .venv/bin/activate
celery -A app.workers.soap_worker.celery_app worker --loglevel=info
```

## Testing

### Test the API is running
```bash
curl http://localhost:3001/api/v1/health
# Expected: {"status": "ok"}
```

### Test SOAP agent in isolation (no server needed)
```bash
python test_soap_agent.py
```
See `AGENTS.md` for full instructions on testing agents.

### Test case ingestion with a sample protobuf payload
```bash
python scripts/send_test_payload.py
```
This script generates a sample `LeanPayload`, encodes it as protobuf, and POSTs it to `/api/v1/cases/ingest`. Check the Celery terminal — you should see the SOAP job picked up within seconds.

## Environment Variables

| Variable | Required | Example | Description |
|----------|----------|---------|-------------|
| `DATABASE_URL` | Yes | `postgresql+asyncpg://user:pass@localhost:5432/medireach` | Async database URL |
| `SYNC_DATABASE_URL` | Yes | `postgresql://user:pass@localhost:5432/medireach` | Sync URL for Celery workers |
| `REDIS_URL` | Yes | `redis://localhost:6379/0` | Redis for Celery broker |
| `GOOGLE_API_KEY` | Yes | `AIza...` | Google AI API key for Gemini and ADK |
| `CLOUD_LLM` | No | `gemini-2.0-flash` | Override the model used by ADK agents |
| `JWT_SECRET` | Yes | any 32+ char string | Signs authentication tokens |
| `DASHBOARD_URL` | Yes | `http://localhost:3000` | Allowed CORS origin |
| `ENVIRONMENT` | No | `development` | Set to `production` to disable debug logging |

## Common Errors

**`ModuleNotFoundError: No module named 'app'`**
You are running the command from the wrong directory. Always run from `apps/api/`.

**`sqlalchemy.exc.OperationalError: could not connect to server`**
PostgreSQL is not running. Run `docker-compose up -d postgres`.

**`google.api_core.exceptions.ResourceExhausted: 429`**
You hit the Gemini free tier rate limit (15 requests/minute). Wait a minute and retry. During development, add `time.sleep(5)` between test calls.

**Celery worker shows no tasks being picked up**
Redis is not running. Run `docker-compose up -d redis`. Also check `REDIS_URL` in `.env`.

**`triage_pb2 not found` or import error on proto file**
You haven't generated the protobuf bindings. Run:
`protoc --python_out=apps/api/app/proto/ proto/triage.proto` from the project root.
